PROBLEM SOLVING

PROBLEM BLOCK

I was blocked on multiple of the Javascript problems from the Katas. I would run tests and get partial fails. Usually if stuck I begin to investigate. I used the methods outlined in the student handbook of looking up how to: ____. I play around until I find a solution that works.

It is important to me to be able to breakdown and explain parts of how the Javascript code functions so at the end of most sprints I would write out the code in notation and annotate Esch part of the code and what it did. I used this method for journal papers at university and while it is a slow learning method, I found it immensely helpful to break things down into smaller parts.

I have often learnt more in depth by going backwards through problems and by being able to explain each piece. I feel I do not understand what I am doing unless I can explain it to someone else. This makes me more competent.

Breaking things down has made me feel more confident in some methods of writing javascript but some still I have not wrapped my head around. Repetition and practice will help.

ELEGANTLY SOLVED A PROBLEM

This simply has not happened yet. I do not feel I have enough intuitive knowledge to solve Javascript problems without referring to external material.
