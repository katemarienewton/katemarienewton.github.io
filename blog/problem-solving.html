THE DOM

The Document Object Model (DOM) is like a blueprint of your webpage. It’s a structured layout that JavaScript can read and modify. Imagine your webpage is a house. The DOM is like the wiring and plumbing inside — you can switch lights on, adjust water flow, or change room layouts by interacting with it.

ARRAYS VS OBJECTS

Array is like a numbered list, where access to array items is done by their position (starting at 0). 

      let fruits = ["apple", "banana", "cherry"]

Object’s are like labelled boxes where object data can be accessed by its label. 

      let person = { name: "Alex", age: 30, job: "Teacher" }

Key Difference: Arrays are best for ordered lists, while objects are better for grouped information with labels.

FUNCTIONS

A function is like a recipe, a set of steps you can reuse whenever you need. 

      function makeCoffee(type) {

        console.log("Boil water");
        if (type === "espresso") {
            console.log("Add espresso shot");
        } else if (type === "latte") {
            console.log("Add espresso shot and milk");
        } else {
            console.log("Add instant coffee");
        }
        console.log("Pour water and stir");
    }
